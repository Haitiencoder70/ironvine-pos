# Database Documentation

The T-Shirt POS Database relies heavily on PostgreSQL handled via Prisma ORM for type-safe guarantees.

## Tenancy
The schema is designed as a Multi-tenant model via logical separation. Every primary data tier (Orders, Inventory, Customers, Shipments, etc.) requires an explicit `organizationId` foreign key.

## Primary Entities
1. **Organization**: The tenant boundary representing a single T-Shirt shop. Contains settings, stripe customer records, and tax configurations.
2. **User**: Clerk ID references defining the actual logged-in actors bounded to an organization.
3. **Customer**: Sourced targets that submit orders internally mapped cleanly natively.
4. **Order**: Core functional pipeline structure matching arrays of `OrderItem` mappings linking sizes, print locations, and design notes natively. Validates against `OrderStatusHistory` capturing explicit user actions over workflows.
5. **InventoryItem**: Standard tracked units (shirts, transfers, vinyl).
